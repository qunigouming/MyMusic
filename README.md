# 项目概述
MyMusic 是一款基于分布式服务器架构的音乐播放系统，旨在提供稳定、高效的音乐播放体验。系统分为客户端与服务器端两部分，客户端聚焦于友好的用户交互与本地音乐管理，服务器端采用分布式设计保障服务可靠性，整体技术栈融合了跨平台 UI 框架、多媒体处理库及分布式服务技术。
使用Visual Studio 2019进行开发，Qt版本6.5.3，客户端仿照网易云音乐界面，实现了基本的登录，注册，播放等功能。项目部分三方库采用vcpkg管理。Boost,grpc,mysql-connector,libjson等需自行安装。

# 核心功能

## 客户端功能
### 1. 用户交互基础功能
已实现用户登录、注册功能，支持通过邮箱验证码完成身份验证，同步展示个性化用户信息（头像、用户名等）。
### 2. 音乐播放与管理
- **本地音乐播放**，基于 FFmpeg 实现音频解码，通过 Qt 的 IODevice 输出声音（预留视频解码扩展能力）。
- **在线音乐播放**：新增在线音乐播放功能，支持连接服务器获取音乐资源并实时播放。
- **歌单功能**：可收藏歌曲放入歌单，客户端可连接服务器获取歌单歌曲功能，同时可播放。
- **本地音乐扫描**：采用多线程文件扫描工具，快速检索指定目录下的所有.mp3 文件，并通过 taglib 解析音乐元数据（标题、歌手、专辑、时长等）。
- **自定义音乐列表**：通过重写 QTableView、Model、Delegate、ProxyModel 及 HeadView，实现个性化音乐列表展示样式。

### 3. 交互体验优化
- 支持皮肤颜色自定义，可根据用户偏好调整界面色彩方案。
- 实现窗口拖拽、最小化 / 最大化 / 关闭等基础窗口管理功能，以及心跳机制维持服务器连接。
## 服务器端功能

### 1. 分布式架构
服务器端包含验证服务、会话服务、网关服务等多个功能模块，通过 gRPC 实现服务间高效通信，保障系统扩展性与稳定性。

### 2. 数据管理
- 维护用户信息、音乐收藏状态等数据，支持歌单信息管理（包含标题、封面、描述、作者、歌曲数量等属性）。

- 依赖 Redis（通过 ioredis 客户端）实现高效数据存储与访问。

# 技术栈
## 客户端
- **UI 框架**：Qt（用于界面构建、事件处理、窗口管理）。
- **多媒体处理**：FFmpeg（音频解码）、taglib（MP3 元数据解析）。
- **线程与并发**：Qt 多线程（本地音乐扫描）。
- **网络通信**：TCP 协议（客户端与服务器数据交互）Http 协议（登录界面时的通信）、心跳机制维持连接。

## 服务器端
- **服务架构**：分布式服务（验证服务、会话服务等）。
- **通信协议**：gRPC（服务间通信）。
- **数据存储**：Redis（依赖 ioredis 客户端）。

## 特色亮点
1. **跨技术栈整合**：融合 Qt 客户端开发、FFmpeg 多媒体处理、分布式服务与 gRPC 通信，实现功能闭环。
2. **高效本地音乐管理**：多线程扫描与元数据解析结合，兼顾扫描速度与信息完整性。
3. **可扩展性设计**：服务器端分布式架构与客户端预留的视频解码功能，为后续扩展提供空间。
4. **个性化交互**：自定义音乐列表样式与皮肤颜色调整，提升用户体验。
